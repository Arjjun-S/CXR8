1. PROJECT OVERVIEW
================================================================================

This is a structured research pipeline for multi-label thoracic disease 
classification using the CXR8 (NIH Chest X-ray) dataset.

OBJECTIVE:
Train and compare multiple deep learning architectures on chest X-ray 
classification, with proper handling of class imbalance and reproducible 
experiment tracking.

MODELS IMPLEMENTED:
  1. Vision Transformer (ViT-B/16)  - Transformer-based, attention mechanism
  2. Swin Transformer (Swin-T)      - Hierarchical, shifted windows
  3. ResNet50                       - CNN baseline, residual connections
  4. DenseNet121                    - CNN, dense connections (CheXNet baseline)

KEY FEATURES:
  - Patient-wise train/val split (prevents data leakage)
  - Class-weighted loss for imbalance handling
  - Medical-appropriate augmentations
  - GPU-only training (30 epochs max)
  - Standardized metrics across all models
  - Automatic saving of results and visualizations

7. METRICS AND EVALUATION
================================================================================

REPORTED METRICS:
-----------------

1. ACCURACY
   Definition: (TP + TN) / (TP + TN + FP + FN)
   Use: Overall correctness, but can be misleading with imbalance

2. PRECISION (Macro-averaged)
   Definition: TP / (TP + FP) averaged across all classes
   Use: How many predicted positives are actually positive
   
3. RECALL / SENSITIVITY (Macro-averaged)
   Definition: TP / (TP + FN) averaged across all classes
   Use: How many actual positives are correctly identified
   Critical for medical diagnosis (catching all disease cases)

4. SPECIFICITY (Macro-averaged)
   Definition: TN / (TN + FP) averaged across all classes
   Use: How many actual negatives are correctly identified

5. F1-SCORE (Macro-averaged)
   Definition: 2 * (Precision * Recall) / (Precision + Recall)
   Use: Harmonic mean of precision and recall

6. ROC-AUC (Macro-averaged, OvR)
   Definition: Area under ROC curve, averaged across classes
   Use: Classification performance across all thresholds
   Range: 0.5 (random) to 1.0 (perfect)

WHY MACRO-AVERAGING?
--------------------
Macro-averaging treats all classes equally regardless of sample count.
This is important for medical imaging where minority classes (rare diseases)
are clinically just as important as majority classes.

8. CLASS IMBALANCE NOTES
================================================================================

THE PROBLEM:
------------
CXR8 dataset has severe class imbalance:
  - No_Finding: Very high count (majority class)
  - Hernia: Very low count (minority class)

Imbalance ratio can exceed 50x between majority and minority classes!

HOW WE HANDLE IT:
-----------------

1. CLASS-WEIGHTED LOSS
   Formula: weight[c] = total_samples / (num_classes * samples_in_class[c])
   Effect: Increases loss contribution from minority classes

2. LABEL SMOOTHING (0.1)
   Effect: Prevents overconfident predictions, helps minority classes

3. CAPPED SAMPLING
   Each class limited to 5,000 images maximum to reduce imbalance

4. MACRO-AVERAGED METRICS
   Reports performance equally weighted across all classes

5. PATIENT-WISE SPLIT
   Prevents data leakage where same patient appears in train and val

RECOMMENDATIONS FOR SEVERE IMBALANCE:
-------------------------------------
If imbalance ratio > 10x:
  - Consider focal loss instead of CrossEntropy
  - Use oversampling of minority classes
  - Report per-class metrics in addition to macro-averaged
  - Consider binary classification per disease (one-vs-all)

-------------------

  - GPU device used

---------------------
